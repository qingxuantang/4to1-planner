### ğŸš€ How to Submit Contributions

#### For Bug Reports:
```markdown
**Title:** Brief description of the bug

**Description:**
- What happened?
- What did you expect?
- Steps to reproduce

**Environment:**
- PDF viewer/printer used
- Paper size (A4/A5)
- Any relevant screenshots
```

#### For Feature Suggestions:
```markdown
**Title:** Feature name

**Description:**
- What problem does it solve?
- How would it work?
- Who would benefit?

**Mockups/Examples:**
- Include sketches or examples if possible
```

#### For Code/Template Contributions:

1. **Fork the repository**
2. **Create a branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes**
   - Follow existing file naming conventions
   - Maintain bilingual support (EN/CN) where applicable
   - Ensure proper print bleed settings for PDFs
4. **Test thoroughly**
   - Print test pages
   - Check alignment and formatting
5. **Commit your changes**: `git commit -m "Add: Brief description"`
6. **Push to your fork**: `git push origin feature/your-feature-name`
7. **Open a Pull Request**
   - Clearly describe what you changed and why
   - Reference any related issues
   - Include before/after screenshots if applicable

### ğŸ“ Contribution Standards

**File Naming:**
- Use descriptive English names
- Format: `Component Name.pdf` or `Component Name Cover.pdf`
- Be consistent with existing files

**Design Standards:**
- Maintain the 4-color system (Blue/Green/Orange/Red)
- Use standard fonts available on most systems
- Ensure 300 DPI for print quality
- Include 0.125" bleed on all sides for professional printing
- Test on both A4 and Letter paper sizes when possible

**Documentation:**
- Write in clear, simple language
- Provide examples where helpful
- Maintain bilingual content (EN/CN) for main documents

**Commit Messages:**
- Start with a verb: Add, Fix, Update, Remove, Refactor
- Be concise but descriptive
- Example: "Add Spanish translation for 4-Year Plan template"

### ğŸŒ Translation Guidelines

We especially welcome translations! To contribute a translation:

1. **Choose a component** (e.g., 4-Year Plan, To-Do List)
2. **Create a new file**: `[Component Name] - [Language Code].pdf`
   - Examples: `4-Year Plan - ES.pdf` (Spanish), `To-Do List - JA.pdf` (Japanese)
3. **Translate all text elements**
   - Maintain the same layout and structure
   - Ensure cultural appropriateness
   - Keep formatting consistent
4. **Add language info to README**
5. **Submit a Pull Request**

**Priority Languages:**
- Spanish (ES)
- French (FR)
- Japanese (JA)
- German (DE)
- Portuguese (PT)

### ğŸ† Recognition
